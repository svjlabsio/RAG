# DocSage

A portfolio RAG (Retrieval-Augmented Generation) application built for technical audiences. Demonstrates a production-grade hybrid search pipeline with full transparency — every SQL query, embedding decision, and ranking step is exposed in the UI.

**Stack:** Streamlit · Neon PostgreSQL · pgvector · tsvector · sentence-transformers · Claude claude-haiku-4-5-20251001

---

## How It Works

Documents are parsed, chunked, and embedded into a single PostgreSQL database. Queries run two searches in parallel — cosine ANN via pgvector and BM25 full-text via tsvector — then merge the results with Reciprocal Rank Fusion before sending the top 5 chunks to Claude.

```
Upload → parse → chunk (512 chars, 50 overlap) → embed (384-dim) → store
Query  → embed → vector search + BM25 → RRF merge → Claude → answer
```

For a full deep-dive into every design decision, see [ARCHITECTURE.md](ARCHITECTURE.md).

## Quick Start

**1. Clone and install dependencies**

```bash
git clone <repo-url>
cd RAG
pip install -r requirements.txt
```

**2. Configure environment**

```bash
cp .env.example .env
# edit .env and fill in DATABASE_URL and ANTHROPIC_API_KEY
```

```env
DATABASE_URL=postgresql://user:password@ep-xxx.neon.tech/neondb?sslmode=require
ANTHROPIC_API_KEY=sk-ant-...
```

**3. Apply the database schema**

```bash
psql $DATABASE_URL -f db/schema.sql
```

Or via Python if `psql` is not available:

```python
import os, psycopg2
from dotenv import load_dotenv
load_dotenv()
conn = psycopg2.connect(os.environ["DATABASE_URL"])
conn.autocommit = True
with open("db/schema.sql") as f:
    conn.cursor().execute(f.read())
```

**4. Run the app**

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501).

## Using the App

### Ingest tab

Upload a PDF, TXT, MD, or DOCX file — or click one of the three sample document buttons. After ingestion the app shows:

- Chunk count, embedding time, and DB insert time
- **Suggested questions** generated by Claude based on the document content
- An expandable panel showing the actual chunking source code and SQL schema

### Query tab

Type a question (or use one of the suggested ones from the ingest step). The app shows:

- The answer from Claude
- Token counts and latency
- An expandable panel showing the exact vector SQL, BM25 SQL, RRF formula and source code, and the full prompt sent to Claude
- A table of the retrieved chunks with vector score, BM25 rank, and RRF score per chunk

### Sample questions to try

After loading the bundled sample documents:

| Question | Source |
|---|---|
| What is retrieval-augmented generation? | rag_guide.md |
| Why does RAG beat fine-tuning for dynamic knowledge? | rag_guide.md |
| Explain the CAP theorem | system_design.txt |
| How does consistent hashing reduce data redistribution? | system_design.txt |
| How does Raft consensus elect a leader? | distributed_systems.txt |
| What are CRDTs and why do they guarantee convergence? | distributed_systems.txt |
| What causes write amplification in LSM trees? | distributed_systems.txt |
| What is the capital of France? | → "I don't have enough context..." |
| CAP *(single acronym)* | BM25 rank populated — vector search alone would miss this |

The last two are intentional: the first tests graceful refusal, the second demonstrates the hybrid search advantage over pure vector search.

| Variable | Description |
|---|---|
| `DATABASE_URL` | Neon PostgreSQL connection string — must include `sslmode=require` |
| `ANTHROPIC_API_KEY` | From [console.anthropic.com/settings/keys](https://console.anthropic.com/settings/keys) |

The embedding model (`all-MiniLM-L6-v2`) and Claude model (`claude-haiku-4-5-20251001`) are constants in `rag/ingest.py` and `rag/generation.py` respectively — change them there if needed.

## Design Highlights

- **Single database** — pgvector for ANN + tsvector for BM25, no separate vector DB
- **Hybrid search** — cosine similarity catches semantic paraphrases; BM25 catches exact terms and acronyms that aren't geometrically close in embedding space
- **RRF fusion** — merges two incommensurable score scales without calibration using the Cormack et al. (2009) formula
- **Local embeddings** — `all-MiniLM-L6-v2` runs on CPU with no API cost
- **Thread-safe singletons** — connection pool, embedding model, and Claude client each use double-checked locking

See [ARCHITECTURE.md](ARCHITECTURE.md) for the full technical rationale behind each decision.
